Recursion

In this lesson you will learn the concept of recursion and how to apply it in Java.

Introduction

Recursion is a concept in computer programming that generally refers to a method, function, or procedure that will continuously call itself until a specific case is met. This can be very useful in situations such as tree traversals, Depth First Search, etc. Many problems can be solved quite easily using recursion.

Base Case

The first part of a recursive function is the base case. This is the case that, if true, will end the recursion and not call the recursive function again.

public int factorial(int n) {
          if (n <= 1) //The base case
            return 1;
          return n*factorial(n-1);
        }

In the above example, the base case is n<=1. If this base case is false, it recursively calls itself but decreases n by 1. This means that n will keep shrinking until n is less than or equal to 1. This method is a simple implementation of factorial. Factorial in math is basically taking every number from 1 to that number and multiplying it. For example, 5! (the exclamation mark means factorial), would be equivelant to 1×2×3×4×5 which is 120. This can be simplified into sets of smaller problems. 5! can be simplified to 5×4! which can be simplified to 5×4×3! etc. This is the basic concept of recursion.

Stack Overflow Error

One very common error that occurs when using recursion is Stack Overflow. This means that the stack, which is the list of methods being called, has grown too big for the computer to handle. This often occurs in recursion when the base case has been written incorrectly and therefore is never reached. This causes a Stack Overflow error because the function will continuously call itself forever since the base case is never met.

Advantages & Disadvantages

Every recursive program can be writter iteratively with a loop, and vice versa. However, there are advantages and disadvantages to both strategies. Disadvantages of recursive programming include a much larger space requirement since each function will remain in the stack until the base case is reached, and it has greater time requirements as well due to the function calls and delay time of returning from a function. However, recursive prgoramming also has many advantages. Recursion provides a much cleaner and simpler way of righting code to solve these types of problems. Some problems are more optimal for recursive solutions such as tree traversals, however even these can be solved with iterative solutions as well.
