Sorting Arrays & ArrayLists

In this lesson you will learn how to sort an Array and an ArrayList in Java.

Introduction

Sorting an array is often necessary in many programs. In this lesson you will learn one basic but efficient algorithm for sorting an array, called Comb Sort. This algorithm is very basic and should not be used in more complex programs. There are many other sorting algorithms that are much more efficient that are not covered in the scope of this lesson. Comb Sort is a pretty basic algorithm for sorting data sets that is much easier to understand than some other, more complex algorithms. Comb Sort is mainly an improvement over Bubble Sort, however, since Bubble Sort is not covered in this lesson, I will assume no knowledge of Bubble Sort when explaining Comb Sort.

How it Works

Comb Sort works by comparing pairs of values and swapping them if they need to be swapped. Instead of comparing adjacent values like Bubble Sort, Comb Sort uses a gap larger than 1. The gap begins at a large value, depending on the size of the array, and then decreases by a factor if 1.3 each iteration until it reaches 1. The factor 1.3 was observed to be optimal after tests on over 200 000 random data sets.

Implementation

Here is an example of how one would implement Comb Sort in Java.

public class CombSort {
  //Used to calculate the next gap size
  public int getNextGap(int gap) {
    gap = (gap*10)/13;
    if (gap < 1)
    gap = 1;
    return gap;
  }

  //Sorts the array using Comb Sort
  public void sort() {
    int[] arr = {23,6,2,12,67,21,46,32,5,9};
    int gap = arr.length;
    boolean swapped = true;

    while(gap != 1 || swapped) {
      gap = getNextGap(gap);
      swapped = false;

      for (int i = 0; i < arr.length-gap; i++) {
        if (arr[i] > arr[i+gap]) {
          int tmp = arr[i];
          arr[i] = arr[i+gap];
          arr[i+gap] = tmp;
          swapped = true;
        }
      }
    }
  }
}

When to Use

This algorithm's worst time complexity is O(N2) and the best case is O(N). This algorithm is best used when the data is already nearly sorted. Comb Sort is least efficient when the data is completely random or when the data is in reversed order. The size of the data set does not effect the efficiency of this algorithm on nearly sorted data sets, however, on data sets that are not nearly sorted, Comb Sort is much more efficient on smaller data sets.
