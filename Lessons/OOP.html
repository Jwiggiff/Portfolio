OOP

In this lesson you will learn the basics of Object Oriented Programming in Java.

Introduction

Object Orient Programming (OOP) is a way of programming in a more organized way using reusable blocks of code, like templates.

Classes & Objects

A class in Java is a user defined blueprint from which objects can be created. It represents a set of methods and/or properties that all objects of some type must have. Objects are the products that are made from these class blueprints. In a class, a constructor is used to create an object of that class. Constructors can require parameters as well that will be set when an object is created. For example, a dog class would take parameters such as the breed and the name of the dog.

Methods & Instance Variables

Classes contain methods and instance variables that all objects of that type will contain. Methods represent behaviours or actions that can be taken, while instance variables or properties represent information about the object. For example, a dog class may contain methods such as bark, eat, and poop, and properties such as name, breed, weight, and age.

Public VS. Private

Modifiers are used to change which parts of a program can access specific properties or methods of a class. Modifiers are written before the type of a variable or return type of a method. For example:

private String str = "Hello, World!";
public double num = 45.3;

Encapsulation is a way of hiding the properties of a class from all other classes. This prevents unauthorized and incorrect manipulation of properties. Encapsulation can be achieved by declaring all of the instance variables of a class as private, and then create public set and get methods that can change and access these variables.

Static Methods & Variables

static is a keyword that can be used in Java to make methods or properties belong to the class itself instead of the object. By default, methods and properties defined in a class are not accessible until an object is created of that class. Then, these methods and properties can be accessed using the object's name. With static, however, you can make methods and properties accessible before an object is created. You can use either the class name or an object name of an object has been created to access these methods and properties, however it is best practice to use the class name to access static variables. Furthermore, static variables are shared between all objects of a type. This means that if I were to define two objects of the same type, and change a static variable in one of the objects, it would change in the other object as well.

Inheritance

Classes can also extend other classes, known as superclasses. This allows a programmer to create a class hierarchy. For example, you might have a superclass Animal and then have two subclasses, Dog and Cat that both extend Animal. This can be very helpful when you want to have a bunch of classes that all have shared methods and variables. You can use inheritance using the extends keyword as shown below.

class Animal {...} //The superclass
class Dog extends Animal {...}  //A subclass of Animal
class Cat extends Animal {...} //Another subclass of Animal

Abstract Classes & Methods

Abstract classes are a form of superclasses. Abstract classes contain the absolute barebones of a superclass. Abstract classes define the properties just like a normal class, but the methods are only defined by their signature. Abstract methods, which are methods within an abstract class, contain no body. They instead rely on being implemented in the subclasses. Unlike superclasses, an object cannot be created from an abstract class by itself. An abstract class must be extended for it to be used. Note that not all methods in an abstract class need to be abstract

abstract class Animal {
    abstract void makeSound();
    abstract void eat(String food);

    public int getAge() {
      return age;
    }
  }

class Dog extends Animal {
  void makeSound() {
    /* implementation of the makeSound method */
  }

  void eat() {
    /* implementation of the eat method */
  }
}

Polymorphism

Polymorphism is Java's ability to differentiate between two entities with identical names. Java uses the entities signatures to differentiate between them. Polymorphism in Java is usually in two forms, overloading and overriding. Overloading is when you have multiple methods with the same name but with different signatures. Overriding is when a subclass provides a new implementation of a method that a superclass provides.
